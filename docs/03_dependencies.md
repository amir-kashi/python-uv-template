# Dependency Management

This project uses [`uv`](https://github.com/astral-sh/uv) for dependency management and environment reproducibility.

`uv` replaces traditional tools such as:

- `requirements.txt`
- `venv`
- `pip-tools`
- `poetry` (for dependency management)

It provides faster installs and fully reproducible environments.

---

## Files Overview

### pyproject.toml

This file defines the project's dependencies and metadata.

Example:

```toml
[project]
name = "python-uv-template"
version = "0.1.0"

dependencies = [
    "python-dotenv",
]

[dependency-groups]
dev = [
    "pytest",
    "black",
    "ruff",
    "pre-commit",
]
```

Dependencies are divided into:

* **Production dependencies** → required to run the application
* **Development dependencies** → required only for development, testing, and code quality

---

### uv.lock

This file locks the exact versions of all dependencies.

This ensures:

* Reproducible builds
* Consistent environments across developers and production
* No "works on my machine" issues

This file is automatically generated and updated by:

```bash
uv sync
```

It should be committed to Git.

## Installing Dependencies

### Install all dependencies (development)

```bash
uv sync
```

Installs:

* Production dependencies
* Development dependencies

Use this for local development.

---

### Install production dependencies only

```bash
uv sync --no-dev
```

Installs only production dependencies.

Use this for:

* Docker builds
* Production environments
* CI/CD pipelines

---

## Running Commands in the Environment

Instead of activating a virtual environment manually, use:

```bash
uv run <command>
```

Examples:

Run application:

```bash
uv run python -m app.main
```

Run tests:

```bash
uv run pytest
```

Run pre-commit:

```bash
uv run pre-commit run --all-files
```

---

## Adding New Dependencies

### Add production dependency

Edit `pyproject.toml`:

```toml
dependencies = [
    "python-dotenv",
    "requests",
]
```

Then run:

```bash
uv sync
```

---

### Add development dependency

Edit:

```toml
[dependency-groups]
dev = [
    "pytest",
    "black",
    "ruff",
    "pre-commit",
    "ipykernel",
]
```

Then run:

```bash
uv sync
```

---

## Updating Dependencies

To update dependency versions:

```bash
uv sync
```

To upgrade dependencies to newer compatible versions:

```bash
uv lock --upgrade
uv sync
```

---

## Why uv?

Advantages:

* Extremely fast installs
* Reproducible environments via `uv.lock`
* No manual virtual environment activation required
* Compatible with Docker and CI/CD
* Uses standard `pyproject.toml`

---

## Best Practices

* Always commit `uv.lock`
* Use `uv sync` after pulling changes
* Use `uv sync --no-dev` in production environments
* Never edit `uv.lock` manually
* Add dependencies only via `pyproject.toml`
